<h2>What Is Image Compression?</h2>

<p>
  Image compression is the process of reducing the file size of a digital image. There are two primary methods:
</p>

<ul>
  <li>
    <strong>Lossy compression:</strong> Permanently removes some data (like subtle colour variations) to achieve smaller file sizes. Formats like JPEG and WebP use lossy compression. When done correctly, the quality loss is invisible to the human eye.
  </li>
  <li>
    <strong>Lossless compression:</strong> Reduces file size without discarding any data, perfect for graphics with sharp edges or text. PNG and GIF are lossless formats, but they often produce larger files than lossy equivalents for photographic content.
  </li>
</ul>

<p>
  For web use, <strong>WebP</strong> (with JPEG fallbacks) is the modern standard because it offers superior compression ratios—typically 25–35% smaller than JPEG at the same perceived quality. That means faster downloads, lower data usage for mobile customers, and better Core Web Vitals scores.
</p>

<h2>The SEO Connection: Why Speed Matters for Perth Venues</h2>

<p>
  Google's Core Web Vitals include <strong>Largest Contentful Paint (LCP)</strong>, which measures how long the largest image or text block takes to appear. Unoptimized hero images are the most common cause of poor LCP. If your signature dish photo takes five seconds to load, Google sees that as a bad user experience and may rank your site lower in search results. (For a deeper dive into Core Web Vitals, read our guide on <a href="/blog/core-web-vitals-hospitality-websites-explained-simply">Core Web Vitals for hospitality websites</a>.)
</p>

<p>
  Beyond rankings, speed affects conversions directly. A study by Portent found that <strong>the highest‑converting e‑commerce sites load in 1.9 seconds or less</strong>. For Perth venues, that could mean the difference between a customer booking a table or bouncing to a competitor with a faster site.
</p>

<p>
  Image compression is one of the easiest and most effective ways to improve LCP. By reducing file sizes, you decrease the amount of data that must be downloaded, allowing the browser to paint the page faster. This is especially critical for mobile users in areas with slower 4G coverage, such as Fremantle's waterfront or Northbridge's crowded streets.
</p>

<h2>Common Mistakes Photographers Make</h2>

<p>
  Most professional photographers are trained to deliver the highest possible quality for print, not for the web. This leads to several well‑intentioned but harmful practices:
</p>

<h3>1. Delivering Huge, Uncompressed Files</h3>

<p>
  A typical high‑resolution RAW file can be 30–50 MB. Even after editing, photographers often export JPEGs at 100% quality, resulting in 5–10 MB files. A single hero image that size can single‑handedly push your LCP beyond 5 seconds, triggering Google's "Needs Improvement" flag.
</p>

<h3>2. Using the Wrong File Format</h3>

<p>
  TIFF and PNG are great for print and graphics but terrible for web photos because they produce unnecessarily large files. JPEG (or better, WebP) should be the default for any photographic content on a website.
</p>

<h3>3. Ignoring Responsive Image Sizes</h3>

<p>
  A one‑size‑fits‑all image that's 4000 pixels wide is overkill for a mobile screen. Modern websites should serve different image sizes based on the viewer's device—a technique called responsive images. Photographers who deliver only a single resolution force developers to resize on the fly, often leading to suboptimal compression or layout shifts.
</p>

<h3>4. Over‑Optimizing (The "Pixel‑Pushing" Trap)</h3>

<p>
  On the other end of the spectrum, some photographers compress images so aggressively that details become blurry, colours band, and appetising food looks unappealing. This defeats the purpose of professional photography and can hurt conversion rates just as much as slow loading.
</p>

<h2>Best Practices for Image Compression</h2>

<p>
  Follow these guidelines to ensure your images look great and load fast:
</p>

<ul>
  <li>
    <strong>Format:</strong> Use WebP with JPEG fallbacks. WebP offers superior compression and is supported by all modern browsers.
  </li>
  <li>
    <strong>Dimensions:</strong> Resize images to the maximum display size. Hero images should be no wider than 1920px; body images no wider than 800–900px.
  </li>
  <li>
    <strong>Quality:</strong> Compress JPEG/WebP at 80–85% quality. This reduces file size by 70–80% with virtually no visible loss.
  </li>
  <li>
    <strong>Lazy loading:</strong> Implement <code>loading="lazy"</code> on images below the fold to prioritise above‑the‑fold content.
  </li>
  <li>
    <strong>Descriptive filenames & alt text:</strong> Name files descriptively (e.g., <code>crispy‑pork‑belly‑northbridge‑restaurant.webp</code>) and write alt text that includes local keywords naturally. This helps both SEO and accessibility.
  </li>
</ul>

<h2>Frequently Asked Questions</h2>

<p><strong>What's the difference between lossy and lossless compression?</strong></p>
<p>Lossy compression removes some data to achieve smaller file sizes; lossless compression reduces file size without any data loss. For web photos, lossy compression (JPEG, WebP) is preferred because the quality loss is unnoticeable at appropriate settings.</p>

<p><strong>How do I know if my images are too large?</strong></p>
<p>Run your site through Google PageSpeed Insights or Lighthouse. If LCP is flagged as "Poor" and the report mentions "image optimization," your images are likely too large. Our mini‑audit provides a detailed breakdown.</p>

<p><strong>Can I compress images after they're already on my website?</strong></p>
<p>Yes. You can use tools like Squoosh, ImageOptim, or plugins (for WordPress) to compress existing images. However, for best results, it's better to re‑export from the original high‑resolution files with the correct dimensions and quality settings.</p>

<p><strong>Do I need to use WebP if my site is built on WordPress?</strong></p>
<p>WebP is recommended but not mandatory. Many WordPress plugins (e.g., ShortPixel, Imagify) can automatically convert uploaded images to WebP and serve them to supported browsers. If you're on a custom Astro site, we bake WebP conversion into the build process.</p>
